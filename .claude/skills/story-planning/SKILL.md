# 스토리 기획 스캐폴딩 스킬

**narrative-writer 에이전트와의 차이:** 이 스킬은 절차/템플릿(어떤 구조로, 어디에 쓸지)을 제공하는 **가벼운 스캐폴딩 도구**다. 깊은 다회차 창작 논의나 실제 대사/시놉시스 집필이 필요하면 이 스킬 사용 후 **narrative-writer 에이전트에게 넘긴다.**

**활성화:** "스토리 구조 잡아줘", "플롯 옵션 뽑아줘", "마을 서사 연결 정리해줘" 등.

**절차:**
1. `docs/story/plot-hooks.html`의 미확정 항목(메인 플롯 동기, 마을 간 연결 구조, 엔딩 구조, 빌런 구조, 회수되지 않은 콘텐츠 처리, 텍스트 볼륨)을 먼저 확인한다.
2. 다루려는 항목에 대해 **최소 2~3개 옵션**을 다음 형식으로 뽑는다 — 절대 하나로 확정하지 않는다:
   ```
   옵션 A: <핵심 아이디어 한 줄>
   - 장점:
   - 단점/리스크:
   - 스코프 영향 (docs/risks.html #1과 연동):
   - 옵트인 로맨스/최적해법형 난이도 철학과의 정합성:
   ```
3. 옵션들을 `docs/story/plot-hooks.html`의 해당 항목 아래에 추가하고 (html 태그 구조 유지), 사용자가 고르면 open-question-facilitator 스킬/에이전트를 통해 "확정" 처리한다 (이 스킬 자체는 확정 권한 없음).
4. 마을별 세부 시놉시스가 필요해지면 `new-village-doc` 스킬로 넘긴다 (해당 스킬의 산출물 형식은 별도 미확정 — new-village-doc 스킬 문서 참고).
5. 대사 실작성이 필요하면 `dialogue-writing` 스킬 또는 narrative-writer 에이전트로 넘긴다.
