# 플레이테스트 기록 스킬

**목적:** 반복 테스트가 필요하다고 명시된 항목(보스 밸런싱, 노숙 페이스 등)의 실제 테스트 결과를 흩어지지 않게 `docs/playtest/<날짜>-<주제>.md`에 기록한다.

**활성화:** "플레이테스트 기록", "피드백 정리", "테스트 결과 남겨줘" 등.

**표준 구조:**
```markdown
---
date: <YYYY-MM-DD>
topic: <예: 사막 오아시스 보스 난이도>
related: [../risks.md, ../systems/adventure-combat.md]
---

# <날짜> <주제> 플레이테스트

## 테스트 조건
(빌드 버전, 특산물 아이템 보유 여부 등)

## 관찰 결과
-

## 시사점
(밸런스 조정 필요 여부, 리스크 문서 갱신 필요 여부)

## 후속 조치
- [ ] docs/risks.html 상태 갱신 필요 여부
- [ ] docs/balance/ 수치 조정 필요 여부
```

**규칙:**
- 기록은 사실/관찰 위주로 남기고, 그 자리에서 밸런스 수치를 바로 확정하지 않는다 — 결론은 systems-designer 에이전트와 별도 논의 후 `docs/balance/`에 반영한다.
- 같은 주제로 반복 테스트가 쌓이면 `docs/playtest/README.md`(또는 `.html`)에 주제별 인덱스를 만든다.

**미확정 — 산출물 형식:** `docs/`가 html로 전환됨에 따라 `docs/playtest/<날짜>-<주제>.md`를 `.md`로 유지할지 `.html`로 만들지 아직 결정되지 않았다. 결정 전까지는 사용자에게 먼저 확인한다.
