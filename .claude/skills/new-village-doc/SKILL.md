# 마을 기획 문서 생성 스킬

**목적:** 마을별 시놉시스/설정이 CLAUDE.md에 쌓여 문서가 비대해지는 것을 막고, `docs/villages/`에 일관된 구조로 분리 관리한다.

**활성화:** "마을 추가", "새 마을 기획서", "<마을명> 시놉시스 작성" 등의 요청 시.

**절차:**
1. `docs/villages/<마을명>.md` 파일이 이미 있는지 먼저 확인한다. 있으면 덮어쓰지 말고 기존 내용을 기준으로 이어서 보완할지 확인한다.
2. 없으면 아래 템플릿으로 새로 생성한다:
   ```markdown
   ---
   status: draft
   related: [../systems/villages.md, ../story/plot-hooks.md]
   ---

   # <마을명>

   ## 한 줄 컨셉
   (미정)

   ## 특산물 체계
   - 고유 작물:
   - 가공품:
   - 보스 공략과의 연결:

   ## 던전 테마
   - 이름/컨셉:
   - 특산물 없이 클리어 시 난이도 가중 방식 (docs/systems/adventure-combat.md 참고):

   ## 결혼 가능 NPC 계열
   - 계열:
   - 컴패니언 NPC (1~2명):

   ## 시놉시스 초안
   (narrative-writer 담당 — 미확정 상태로 옵션 제시)

   ## 환경 스토리텔링 요소
   (대사량 대신 풍경/유적/아이템 설명으로 전달할 요소)

   ## 미확정 — 상의 필요
   -
   ```
3. `docs/systems/villages.html`의 마을 표에 해당 마을이 없으면 행을 추가하고, 있으면 이 문서로의 링크를 붙인다 (html 태그 구조 유지).
4. CLAUDE.md에는 절대 직접 내용을 추가하지 않는다 — CLAUDE.md는 인덱스 표만 유지한다.
5. 시놉시스/설정 내용을 임의로 확정하지 말고, 핵심 디자인 철학(CLAUDE.md)과 상충하는지 game-designer 관점에서 한 번 점검한 뒤 "미확정 — 상의 필요" 섹션에 옵션으로 남긴다.

**미확정 — 산출물 형식:** `docs/`가 html로 전환되어, 위 템플릿(markdown)을 그대로 `.md`로 쓸지 `.html`로 만들지는 아직 결정되지 않았다 (향후 문서 편집을 손으로 html 태그를 고치는 방식으로 갈지, md로 작성 후 변환하는 방식(`docs-html-export` 패턴 부활)으로 갈지에 달림). 결정 전까지는 사용자에게 이 문서를 `.md`/`.html` 중 무엇으로 만들지 먼저 확인한다.
