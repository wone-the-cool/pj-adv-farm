# 밸런싱 표 작성 스킬

**목적:** CLAUDE.md의 "참고할 만한 스킬/툴 방향 — 스프레드시트 스킬"을 실제로 수행. 수치성 밸런싱 데이터를 CLAUDE.md/docs 산문에 섞지 않고 `docs/balance/`에 표(CSV 또는 마크다운 표) 형태로 분리 관리한다.

**활성화:** "작물 가격/희귀도 표", "보스 난이도 곡선", "경험치 곡선", "밸런싱 시뮬레이션" 등의 요청 시.

**절차:**
1. 대상 파일 위치: `docs/balance/<주제>.csv` (예: `docs/balance/crop-prices.csv`, `docs/balance/boss-difficulty.csv`, `docs/balance/xp-curve.csv`). 처음이면 `docs/balance/` 디렉터리와 함께 생성한다.
2. 표 컬럼은 항목마다 다르지만 최소한 다음을 포함한다:
   - 작물 가격/희귀도: `마을, 작물명, 기본가격, 희귀도(1-5), 가공품명, 가공품가격, 비고`
   - 보스 난이도: `마을, 보스명, 기본체력, 특산물장비O_데미지배율, 특산물장비X_데미지배율, 특산물장비X_소요시간배율, 비고`
   - 경험치 곡선: `레벨, 필요경험치, 누적경험치, 비고`
3. 수치는 항상 **가정치(placeholder)**로 표시하고, 파일 상단에 주석/헤더로 "실제 개발/테스트 단계에서 조정 예정 — 기획 초기 확정 아님"을 명시한다 (CLAUDE.md 6번 원칙과 동일).
4. 표 생성/갱신 후 관련 `docs/systems/*.html` 문서의 "다음 작업 후보"에 해당 balance 파일 링크를 추가한다 (html 태그 구조 유지).
5. 수치를 확정된 사실처럼 서술하지 않는다 — systems-designer 에이전트 관점에서 "반복 테스트 필요" 문구를 유지한다.
