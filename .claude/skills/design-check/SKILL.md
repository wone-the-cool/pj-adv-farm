# 신규 제안 디자인/스코프 점검 스킬

**목적:** CLAUDE.md "Claude Code 작업 시 참고사항"의 "신규 시스템 제안 시 기존 확정 방향과 상충 여부 점검"을 매번 놓치지 않고 수행하기 위한 체크리스트형 스킬. game-designer와 scope-reviewer 관점을 하나로 묶어 빠르게 1차 점검한다.

**활성화:** 새 시스템/컨텐츠/기능 제안이 나올 때, 또는 사용자가 명시적으로 검토를 요청할 때.

**점검 체크리스트 (순서대로):**
1. **철학 상충 여부** (CLAUDE.md 핵심 디자인 철학 4개 원칙 기준)
   - 농장 경영이 주, 모험은 종이라는 비중을 해치지 않는가?
   - 특정 공략법 없이도 (어렵지만) 클리어 가능한 구조인가, 아니면 완전 자물쇠-열쇠인가?
   - 강제 콘텐츠가 아니라 옵트인 인센티브 구조인가?
   - 스타듀밸리식 여유 페이스("오늘은 농장만 해도 되는 자유")를 해치지 않는가?
2. **스코프 여부** (docs/risks.html #1, docs/roadmap.html 기준)
   - 이 제안이 MVP 범위에 들어가야 하는가, 확장팩으로 미뤄도 되는가?
   - 기존 시스템으로 대체 가능한데 새 시스템을 만들려는 것은 아닌가?
3. **미확정 항목 임의 확정 여부**
   - 이 제안이 `docs/`의 "미확정 — 상의 필요" 항목을 건드리는가? 그렇다면 결정하지 말고 옵션+트레이드오프로 제시했는가?
4. **문서화 위치**
   - 세부 내용이 CLAUDE.md에 직접 추가되고 있지 않은가? (있다면 관련 `docs/systems/`, `docs/story/`, `docs/format/` 등으로 옮길 것)

**출력 형식:** 위 4개 항목에 대해 O/X/△(조건부)로 짧게 답하고, X 또는 △인 항목만 이유와 대안 옵션을 덧붙인다. 통과 여부를 대신 결정하지 않는다 — 최종 판단은 사용자 몫.
