# NPC 로맨스/컴패니언 - 옵트인 구조 (CLAUDE.md "핵심 시스템 5" 상세본)

## 확정된 원칙
- NPC 상호작용은 메인 콘텐츠가 아닌 별도 재미 요소 — 원치 않는 호감도 관리 부담 최소화가 목표.
- 스타듀밸리식 "선물만 줘도 자동 호감도 상승" 방식 지양.
- **명시적 트리거 방식** 채택: 특정 대화 선택지에서만 호감도 상승, 또는 별도 "구애 모드"를 켜야 호감도 시스템 작동.
- 결혼 가능 NPC는 정착한 마을의 고유 NPC로 한정 → 마을 선택에 의미 부여.
- 결혼과 별개로 필드 탐험 동행 가능한 컴패니언 NPC를 마을별 1~2명 배치, 전투 보조/채집 보너스 등 실질적 이득 제공.

## 미확정 — 상의 필요
- "구애 모드" UI/UX 구체안 (토글 방식? 특정 아이템/대사로 진입?)
- 컴패니언 동행 시스템과 로맨스 시스템의 관계 — 컴패니언이 곧 로맨스 대상인지, 별개 풀인지
- 결혼 후 콘텐츠 설계 (CLAUDE.md 리스크 5번) — 스타듀밸리의 대표적 비판(결혼 후 콘텐츠 부재) 반복 방지
