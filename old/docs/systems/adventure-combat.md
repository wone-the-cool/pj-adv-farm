# 보스/던전 공략 시스템 (CLAUDE.md "핵심 시스템 2" 상세본)

## 확정된 원칙
- 마을 특산물 포션/장비 있으면 훨씬 수월, 없어도 시간·위험 감수 시 클리어 가능해야 함.
- 예: 화염 저항 포션 없이도 불 데미지 감수하며 클리어 가능 / 있으면 훨씬 수월.
- **금지**: 완전 자물쇠-열쇠 구조(아이템 없으면 데미지 자체가 안 박히는 방식) — 선형적 순서 강제로 이어져 자유도 철학과 충돌.

## 미확정 — 수치/구현 필요 (systems-designer 담당)
- "없어도 가능하되 훨씬 어려움"을 구체적으로 어떤 수치로 구현할지: 데미지 배율? 소요 시간? 리소스 소모량?
- 전투 조작 방식(단순 액션 vs 스킬/콤보) — `docs/format/game-format-tbd.md` 참고, 전투 복잡도가 정해져야 난이도 곡선 설계 가능
- 던전별 난이도 곡선, 보스 체력/패턴 설계는 반복 테스트 필요 (CLAUDE.md 리스크 3번)

## 다음 작업 후보

- 난이도 밸런싱 시뮬레이션 스프레드시트
