# 개발 마일스톤/일정 (뼈대 — 미확정)

일반 게임 기획 제안서의 표준 목차 항목 중 하나로 신설. 아직 실제 일정/인력 정보가 없어 구조만 제시하며, 임의로 기간·인력을 확정하지 않는다.

## 제안 마일스톤 구조 (옵션 — 확정 아님)

| 단계 | 목표 | 포함 범위(안) | 산출물 |
|---|---|---|---|
| M0. 컨셉 검증 | 핵심 루프(농장+모험 왕복)가 재미있는지 확인 | 마을 1개, 보스 1종, 최소 농사 루프 | 플레이 가능한 버티컬 슬라이스 |
| M1. MVP | 초기 스코프 마을(1~2개) 완결 | 정착/결혼 제외, 대여 농장까지만 | 플레이테스트용 빌드 |
| M2. 정식 출시 | MVP 마을 + 정착/결혼/엔딩 | 전체 루프 완성 | 출시 빌드 |
| M3. 확장팩 | 나머지 마을 추가 | 신규 마을별 세트(작물/던전/NPC) | DLC/업데이트 |

## 미확정 — 상의 필요
- 팀 규모/1인 개발 여부에 따라 위 단계별 소요 기간이 크게 달라짐 — 개발 리소스 정보 필요
- MVP에 넣을 마을 1~2개를 어떤 기준(서사 대표성/구현 난이도/차별화 포인트)으로 고를지 → docs/systems/villages.md "개발 우선순위" 열과 연동
- 확장팩 단위를 마을 1개씩으로 쪼갤지, 2개씩 묶을지

## 다음 작업 후보
