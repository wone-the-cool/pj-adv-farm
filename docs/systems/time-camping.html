시간 시스템 - 리얼타임 + 노숙 (CLAUDE.md "핵심 시스템 3" 상세본)

확정된 원칙

  • 모험 중에도 시간은 실시간으로 흐름 (마을/필드 간 시간축 분리 없음).
  • 스타듀밸리식 오전 2시 강제 기절/귀가 없음 — 대신 노숙(캠프파이어)으로 필드에서 밤을 지샐 수 있음.
  • 노숙은 선택적 콘텐츠: 자정 전후 특수 몬스터/이벤트, 캠프파이어 동료 상호작용/대사, 야간 전용 채집 아이템 등 보상형 콘텐츠 제공.

제안 — 전역 캘린더 vs 마을별 기후 분리 (2026-07-07, 회의록, 마을1 검증 전까지 제안 상태)

마을2·3이 둘 다 "여름" 컨셉이라 계절 중복 우려가 나왔으나, 사용자가 "시간/계절이 주는 시간제약의 재미는 포기하고 싶지 않다"고 확인하면서 나온 절충 구조:

  • 전역 캘린더(그대로 유지): day counter 1개(세이브값), 축제/보스 등장/날씨/시세 변동 등 시간압박 전담.
  • 마을별 비주얼/생태 정체성(항상 고정): 전역 계절과 무관하게 그 마을 고유 외형(다래계곡=항상 열대, 청파나루=항상 해안절벽) 유지.
  • 마을별 특산작물 재배 시즌표: 그 마을 작물이 전역 캘린더의 어느 시즌 슬롯에서 자라는지는 마을마다 다르게 지정 가능 — 비주얼상 계절과 실제 재배 타이밍이 반드시 일치할 필요 없음(Farming Simulator의 "비주얼 계절 라벨 ≠ 실제 성장 타이머 분리" 선례).
  • 가드레일: 어떤 재배 시즌표도 진행 자체를 봉쇄해선 안 됨 — 시즌을 놓쳐도 상인 구매/무역 등 "더 느리고 비싼 대안"으로 우회 가능해야 함 (기존 확정된 자물쇠-열쇠 금지, 실패 페널티 절대 금지 원칙의 연장).
  • 진저섬처럼 마을2/3을 계절 자체에서 완전히 면제하는 안은 반려 — 두 마을 모두 보스 공략에 필수적인 core 마을이라 후반 보너스 지역과 지위가 다름.
  • 구현/세이브 영향은 낮음: 전역 day counter 1개 + 마을별 climate_profile(정적 콘텐츠 데이터, 세이브 아님) + (작물,마을) 재배가능표. 구체 설계는 마을1 이후 착수.

미확정 — 상의 필요 (CLAUDE.md 리스크 4번)

  • 노숙 보상의 구체적 종류/빈도: 매일 밤 보상이 있으면 사실상 필수화될 위험, 너무 드물면 매력이 없음 → 빈도 밸런스 옵션 필요
  • 강제 없이도 "하고 싶어지는" 유인 설계: 한정 재화? 스토리 전용 이벤트? 컴패니언 호감도(옵트인 로맨스와 연동 여부)?
  • 노숙 중 리스크(몬스터 습격 등)를 얼마나 강하게 걸지 — 너무 약하면 의미 없고, 너무 강하면 페이스를 해침
  • 위 "전역 캘린더 vs 마을별 기후 분리" 제안의 구체 스펙(재배 시즌표 수치 등) — 마을1 검증 후
  • 리텐션 참고자료 기준 뚜렷한 갭: 세션 단위 유연성(자투리 시간 플레이 vs 몰입 플레이 모두 지원하는지) — 미검토, 마을1 검증 이후
