스토리라인 관련 논의점

CLAUDE.md에서 이동. narrative-writer 에이전트가 주로 참고/발전시키는 문서.

확정된 사항 (2026-07-03, 마을 개방 구조 회의록에서 함께 결정)

  • 메인 플롯의 큰 줄기: "흩어진 힘/파편을 모아야 고향을 되찾는다"류 설정 — 자유 순회의 이유이자 공통 빌런 서사의 그릇 역할.
  • 마을 간 서사적 연결고리: 완전 옴니버스가 아니라 공통 빌런의 파편형 구조. 각 마을 보스는 더 큰 존재의 파편이되, BOTW 신수 방식처럼 방문 순서와 무관하게 그 자체로 완결된 이야기가 되도록 집필한다 (전면 개방 정책과 충돌하지 않도록).
  • 엔딩 구조: 정착 마을 선택은 메인 서사 분기가 아니라 에필로그(로맨스/정착 플레이버)만 분기시킨다. 메인 서사는 공통 엔딩으로 수렴 — 분기 스토리라인 다중 관리 비용을 피함.
  • 보스/빌런 설정: 각 마을 보스는 개별 위협이 아니라 더 큰 존재의 파편/하수인.

주의(narrative-writer가 스스로 짚은 트레이드오프): 방문 순서 무관 일관성을 위해 플래그 기반 대사 분기가 필요해 대사 스코프가 늘어난다 — "최소 플래그 언급" 수준으로 제한할 것 (scope-reviewer 견제 대상).

미확정 — 상의 필요

  • 회수되지 않은 마을 콘텐츠 처리: 정착하지 않은 마을들의 서브플롯이나 로맨스 라인을 스토리상 어떻게 자연스럽게 매듭짓거나 열어둘지. (부분 힌트: 전면개방+파편구조라 플레이어가 대부분 마을을 결국 방문하게 되어 "낭비" 문제가 구조적으로 줄어듦 — narrative-writer는 미정착 마을에 짧은 클로징(편지, 축제 카메오 등) 배치를 권장했으나 구체 설계는 미완료)
  • 텍스트 볼륨: NPC 상호작용 비중을 낮게 유지하기로 한 만큼, 스토리 전달을 대사량보다 환경 스토리텔링(마을 풍경, 유적, 아이템 설명 등)에 얼마나 의존할지
