게임 형식 관련 논의점

CLAUDE.md에서 이동. technical-advisor 에이전트가 주로 참고/발전시키는 문서.

확정된 사항

  • 맵/씬 구조 및 잠금 상태 저장 방식 (2026-07-03, 회의록): 맵은 씬 단위로 분리(엔진 무관, 나중에 뒤집기 쉬운 방식). 세이브 스키마 가안: village_id -> {quest_stage, rented_farm_state, specialty_items_collected[], npc_affinity_optin}. "마을 잠금 여부"는 이 값에서 파생되는 계산값으로 처리하고 별도 저장하지 않는다 — 개방 정책(전면/순차)을 나중에 바꿔도 세이브 마이그레이션이 필요 없다.
  • 엔진: Godot (2026-07-03, 회의록): 1인 개발+코딩 가능+PC 선출시 조건에서 결정. MIT 라이선스로 로열티 없음, 네이티브 2D 파이프라인, 1인 반복속도에 유리. 예외 조건: 팜시뮬 전용 프리메이드 에셋(대화 시스템, 인벤토리 등)을 대량 구매할 계획이 생기면 Unity 재검토.
  • 타겟 플랫폼 (2026-07-03, 회의록): Steam(PC) 출시를 우선 목표로 함. 스위치 이식은 가능성 낮음(사용자 확정) — 콘솔 대응을 전제로 한 설계/예산은 배정하지 않는다.
  • 전투 조작 방식 (2026-07-03, 회의록): 기본 2버튼(공격/회피) + 장비(무기 아키타입) 종속 특수버튼. 무기는 검/창/활/도구형 등 3~4종 상한의 "아키타입" 단위로 스킬 로직을 공유하고 마을 특산물 속성만 파라미터로 주입 (예: 도구형+서리 특산물=타격 시 빙결). MVP는 아키타입 1~2종만 우선 구현, 나머지는 확장팩(M3)으로 미룸. 스타듀밸리도 WASD+마우스만으로 전투 조작이 가능하다는 점을 참고해 키보드+마우스로 전체 플레이가 완결되어야 함을 기준으로 삼는다 (게임패드는 추가 지원 여부 별도 논의).

미확정 — 상의 필요

  • 그래픽 스타일: 스타듀밸리 유사 2D 픽셀아트로 확정(스타일 자체는 미확정 목록에서 제외). 다만 해상도/도트 밀도(16x16 vs 32x32 등), 캐릭터 비율(SD형 2등신~3등신 vs 정상 비율)은 여전히 미정.
  • 시점/카메라: 탑다운(스타듀밸리식) 고정, 혹은 쿼터뷰 등 변형 여부 결정 필요.
  • 마을/던전 세부 조작: 전투 조작 큰 틀은 확정됐으나, 마을 내 이동/상호작용, 던전 내 세부 조작(카메라, 상호작용 키 등)은 추가 논의 필요 (사용자 요청으로 보류).
  • 게임패드 지원 여부: 키보드+마우스로 완결되는 것은 확정, 게임패드 추가 지원 여부는 별도 결정 필요.
