게임 형식 관련 논의점

CLAUDE.md에서 이동. technical-advisor 에이전트가 주로 참고/발전시키는 문서.

확정된 사항

  • 맵/씬 구조 및 잠금 상태 저장 방식 (2026-07-03, 회의록): 맵은 씬 단위로 분리(엔진 무관, 나중에 뒤집기 쉬운 방식). 세이브 스키마 가안: village_id -> {quest_stage, rented_farm_state, specialty_items_collected[], npc_affinity_optin}. "마을 잠금 여부"는 이 값에서 파생되는 계산값으로 처리하고 별도 저장하지 않는다 — 개방 정책(전면/순차)을 나중에 바꿔도 세이브 마이그레이션이 필요 없다.
  • 엔진: Godot (2026-07-03, 회의록): 1인 개발+코딩 가능+PC 선출시 조건에서 결정. MIT 라이선스로 로열티 없음, 네이티브 2D 파이프라인, 1인 반복속도에 유리. 예외 조건: 팜시뮬 전용 프리메이드 에셋(대화 시스템, 인벤토리 등)을 대량 구매할 계획이 생기면 Unity 재검토.
  • 타겟 플랫폼 (2026-07-03, 회의록): Steam(PC) 출시를 우선 목표로 함. 스위치 이식은 가능성 낮음(사용자 확정) — 콘솔 대응을 전제로 한 설계/예산은 배정하지 않는다.
  • 전투 조작 방식 (2026-07-03, 회의록): 기본 2버튼(공격/회피) + 장비(무기 아키타입) 종속 특수버튼. 무기는 검/창/활/도구형 등 3~4종 상한의 "아키타입" 단위로 스킬 로직을 공유하고 마을 특산물 속성만 파라미터로 주입 (예: 도구형+서리 특산물=타격 시 빙결). MVP는 아키타입 1~2종만 우선 구현, 나머지는 확장팩(M3)으로 미룸. 스타듀밸리도 WASD+마우스만으로 전투 조작이 가능하다는 점을 참고해 키보드+마우스로 전체 플레이가 완결되어야 함을 기준으로 삼는다 (게임패드는 추가 지원 여부 별도 논의).
  • 도트 밀도/캐릭터 비율 (2026-07-10, 회의록): 월드 타일 그리드 = 16x16(스타듀밸리 방식, 런타임 정수배 업스케일). 캐릭터 캔버스 = 32x64px(가로 2타일×세로 4타일), 비율 = 3.5등신 — 도트아바타 커스터마이징(머리/상의/하의/신발/액세서리 레이어드 파츠) 접근성을 위해 2~3등신에서 상향. 충돌박스는 캔버스 전체가 아니라 발 기준 1타일 이하 feet-box로 분리(스타듀밸리와 동일 방식, 캔버스 크기와 물리 판정 크기는 무관). 화면 배율은 FHD=4x(256px)/QHD=5x(320px)로 해상도별 분리해 체감 크기를 비슷하게 유지한다. 무기 스윙 등 넓은 판정 프레임은 48px 폭 예외를 고려 대상으로 남겨둠(확정 아님, 무기 아키타입 설계 시 systems-designer 재검토). MVP 단계 커스터마이징 파츠 종류 수는 스코프 리스크로 인지하되 구체 개수는 아직 미정.
  • 시점/카메라 (2026-07-13, 회의록): 탑다운(스타듀밸리식 오블리크/앵글드 탑다운) 고정. 쿼터뷰/아이소메트릭은 채택하지 않는다 — 2026-07-10 확정된 정사각 타일 그리드/캐릭터 캔버스 스펙을 그대로 재사용하며(아이소메트릭이었다면 다이아몬드 타일 전환으로 해당 확정 사항이 전부 재작업 대상이 됨), 4방향 스프라이트 기준을 유지한다(8방향 전환 시 애니메이션 프레임이 약 2배로 늘어나는 것을 피함). 모험 파트(보스 정화 연출 등)의 시각적 임팩트 보완은 카메라 줌/이펙트/컷인 초상화 등으로 추후 별도 검토.
  • 마을/던전 세부 조작 (2026-07-13, 회의록): 상호작용은 분리 — "사용"(좌클릭, 소지 아이템에 따라 컨텍스트 자동 판단: 농기구=도구사용/무기=이미 확정된 전투 공격버튼 재사용) / "상호작용"(우클릭 or E, Area2D 근접감지로 대화·줍기·문열기). 조준/타겟팅은 소지 아이템 타입 기준 고정분기 — 농기구=커서방향(정밀), 무기/전투=facing방향. 디바이스별 폴백(둘 다 상시지원) 방식은 채택 안 함. 상호작용 안내는 MVP에서 화면 하단 텍스트 프롬프트만(오브젝트 하이라이트/외곽선은 M1+ 폴리싱 후보로 보류). 마을·던전은 동일 스킴 유지, 던전에서는 전투 레이어(회피+무기특수)가 추가될 뿐 별도 조작 체계를 만들지 않는다.

미확정 — 상의 필요

  • 게임패드 지원 여부: 키보드+마우스로 완결되는 것은 확정, 게임패드 추가 지원 여부는 별도 결정 필요.
