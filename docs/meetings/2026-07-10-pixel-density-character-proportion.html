회의록 — 도트 밀도/캐릭터 비율

안건

docs/format/game-format-tbd.html의 미확정 항목: 해상도/도트 밀도(16x16 vs 32x32 등), 캐릭터 비율(SD형 2~3등신 vs 정상 비율). 픽셀아트 스타일 자체(스타듀밸리 유사 2D)는 이미 확정, 이 세부 스펙만 미정이었음. 2라운드로 진행: 1라운드(월드 타일 그리드 + 초기 비율) → 사용자가 캐릭터 비율을 3.5등신으로 조정 + FHD/QHD 가독성 조건 추가 → 2라운드(캐릭터 캔버스 구체 스펙).

참가

  • game-designer — 장르 정체성/철학 정합성
  • technical-advisor — Godot 구현/렌더링, 캐릭터 캔버스 스펙
  • scope-reviewer — 1인 개발 스코프 비관적 검토

리서치 브리핑 (메인 세션이 직접 WebSearch/WebFetch 수행, 모든 에이전트에게 동일 브리핑 공유)

  • [신뢰도 높음, WebFetch 원문] freegamesprites.com 타일크기 가이드: 16x16=타일당 ~5분(로그라이크/팩토리시뮬/모바일에 최적), 32x32="황금비율"(타일당 15~20분, 현대 인디 2D RPG/액션-어드벤처/메트로이드배니아 표준), 64x64=타일당 45~60분(디지털회화급, 자산팩/멀티팀 권장). 핵심 조언: "타일 크기는 확정 후 변경 금지", "200개 타일 그리기 전 1개로 엔진 테스트 먼저". 출처: freegamesprites.com
  • [신뢰도 높음] ConcernedApe(스타듀밸리 개발자) 확인: 타일은 16x16으로 그려 런타임에 64x64로 4배 업스케일. NPC 오버월드 스프라이트는 16x32(월드 타일보다 큼). 출처: x.com/concernedape, forums.stardewvalley.net
  • [신뢰도 중간, 포럼 종합] 애니메 픽셀아트는 보통 head-to-body 1:2~1:3(치비/SD), 입문자가 1:4~1:5로 그리면 오히려 사실적 비율에 가까워 보임. 출처: gamedev.net
  • [신뢰도 중간] 32x32는 "1인 개발자의 프로급 스위트스팟"으로 자주 언급되나, 근거에 "숙련되면"이라는 전제가 붙는 경우가 많음.

1라운드 — 각 에이전트 입장

에이전트도트 밀도캐릭터 비율핵심 근거
game-designer16x16SD ~3등신NPC 감정표현은 월드스프라이트 도트가 아니라 별도 대화창 초상화(portrait)로 처리하는 게 스타듀밸리/룬팩토리의 검증된 해법 — 애초에 32x32를 선호할 철학적 근거가 약함(자체 교차검토로 초기 입장을 스스로 뒤집음).
technical-advisor16x16 (NPC 16x32)SD 2~3등신저밀도 그리드에선 정상비율(6~8등신)을 애초에 읽을 수 없음 — 밀도가 비율을 사실상 강제. Godot 구현은 texture_filter=nearest + 정수배 업스케일 + snap_2d_transforms_to_pixel. 타일 그리드는 확정 후 되돌리기 어렵지만(전량 재작업), 업스케일 배율/카메라 줌은 순수 정책값이라 언제든 조정 가능.
scope-reviewer16x16SD 2~3등신32x32는 타일당 3~4배 시간 — 마을 최소 3개 비전 + 1인 개발 + 아직 검증 안 된 그래픽 숙련도 조합에서 M0(마을1 검증) 자체를 지연시킬 위험.

1라운드 충돌 지점: 사실상 없음 — 세 에이전트가 서로 다른 결론(특히 game-designer는 로맨스 NPC 개성 표현을 이유로 애초 32x32 쪽에 무게를 두고 시작)을 예상했으나, 실제로는 전부 16x16(월드)+SD 2~3등신으로 수렴. 억지 합의가 아니라 "32x32를 정당화할 근거가 약하다"는 걸 각자 다른 각도(초상화 대체 해법/밀도-비율 강제결합/제작비 3~4배)로 독립적으로 확인.

사용자 조정 (1라운드 결과 검토 후)

"캐릭터 비율은 3~4등신이 좋을거같아 좀더 도트아바타 꾸미기 접근성이 좋게끔. 로블록스도 보면 3d지만 4등신 캐릭터야. 3.5등신 좋을듯." + "FHD, QHD 환경에서 게임했을때 너무 작게만 안보이면 될듯한 사이즈가 좋을듯." — 월드 타일 그리드(16x16)는 유지, 캐릭터 비율만 3.5등신으로 상향 + 아바타 커스터마이징 접근성 + 화면 가독성 조건 추가.

2라운드 — technical-advisor 후속 검토 (캐릭터 캔버스 스펙)

스타듀밸리 파머 스프라이트가 이미 월드 타일(16x16)보다 큰 16x32 캔버스를 쓴다는 선례를 근거로, 환경 그리드와 캐릭터 캔버스를 분리해 구체 스펙 도출.

  • 캐릭터 캔버스: 32x64px(가로 2타일×세로 4타일) — 3.5등신에서 머리/모자/헤어 레이어 구분에 필요한 머리 지름 최소 ~17~18px 확보. 16그리드와 정수배라 스프라이트 아틀라스 슬라이싱 깔끔.
  • 충돌박스는 캔버스 전체가 아니라 발 기준 1타일 이하 feet-box로 분리(스타듀밸리와 동일 방식) — 캔버스가 크다고 물리 판정도 커지는 게 아님.
  • 화면 배율: FHD(1080)=4x(256px, 화면 세로 23.7%) / QHD(1440)=5x(320px, 22.2%) — 해상도별로 다르게 줘야 체감 크기가 비슷하게 유지됨. (대안: 고정 4x로 통일하면 QHD에서 캐릭터는 작아지는 대신 시야(FOV)가 넓어지는 트레이드오프 — 검토했으나 채택 안 함.)
  • 무기 스윙 등 넓은 판정 프레임은 32px 폭을 넘을 수 있어 공격 프레임만 48px 폭 예외 허용.

2라운드 상충/영향: 3.5등신(2~3등신 대비 상향)은 커스터마이징 표현력엔 순이득이지만, 파츠(머리/상의/하의/신발/액세서리) × 애니메이션(이동/도구/전투) 조합의 원화 제작량이 늘어 1인 개발 스코프 부담이 커짐 — scope-reviewer 관점에서 MVP 단계 착용 파츠 종류 수를 의도적으로 제한 권고(구체 개수는 이 회의록에서 확정하지 않음, 후속 논의 필요).

사용자 최종 결정 (2026-07-10)

  1. 월드 타일 그리드: 16x16 확정.
  2. 캐릭터 캔버스: 32x64px(2x4타일), 3.5등신 비율 확정.
  3. 충돌박스: 캔버스 전체가 아닌 발 기준 feet-box로 분리 확정.
  4. 화면 배율: FHD=4x, QHD=5x (해상도별 분리, 체감 크기 유지 우선) 확정.
  5. 공격 프레임 폭 예외(48px)는 "고려 대상"으로만 남기고 이번엔 확정하지 않음 — 실제 무기 아키타입 설계 시 systems-designer가 재검토.
  6. MVP 단계 커스터마이징 파츠 종류 수 제한은 스코프 리스크로 인지만 하고 구체 개수는 미정 — 마을1 검증 단계 이후 실제 파츠 설계 시 확정.
